Tab Structure
탭 구조: 합치기 vs 분리
기존 "일자별 내역" 탭에 흡수할 것인가, 별도 탭으로 분리할 것인가. 두 가지 모두의 네비게이션 모습을 비교합니다.
Option 1: 기존 탭에 흡수 (뷰 전환)
📊 Monitor Daily Analytics
2026-05-19
Sessions = 기존 일자별 내역 (변경 없음)
Changelog = 커밋 기반 성과 뷰 (신규)
Weekly / Monthly = 주간/월간 집계 (신규)
Pro: 탭 수가 늘지 않음. 한 곳에서 세션/성과/집계를 전환.
Con: "Daily" 탭 안에 역할이 너무 많아짐. Sessions과 Changelog는 관점이 다른 데이터. Weekly/Monthly가 Daily 하위에 있는 게 직관적이지 않을 수 있음.
Option 2: 별도 탭으로 분리
📊 Monitor Sessions Daybook Analytics
2026-05-19
Sessions 탭 = 기존 일자별 내역 (100% 보존)
Daybook 탭 = 커밋 기반 성과 + 메모 + 주간/월간 집계
Pro: 역할이 명확. Sessions="세션 원시 데이터", Daybook="정리된 성과 기록". 기존 탭 변경 제로.
Con: 탭이 4개로 늘어남. 같은 날짜 데이터를 다른 탭에서 봐야 하는 중복감.
Option 3: 기존 탭을 대체
📊 Monitor Daybook Analytics
2026-05-19
Daily 뷰 = 성과(커밋) + 세션 목록 합친 형태 (Direction B: Hybrid)
기존 세션 카드 클릭 → SlidePanel로 상세 보기
Weekly / Monthly = 집계 뷰
Pro: 탭 수 유지(3개). 기존 기능 + 신규 기능이 한 곳에. 가장 강력한 업그레이드 느낌.
Con: 기존 일자별 내역의 그리드 레이아웃이 사라짐. 익숙한 UI가 변경됨.
